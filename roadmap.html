MVP Scope

한 달 안에 보일 기능과 나중에 붙일 기능을 분리합니다.

라이브 요구사항은 시청, 채팅 통합, 투표, 시청자 수, 방송자 화면, 권한 분리가 핵심입니다. 후원과 공개 채널은 라우터만 먼저 열어둘 수 있습니다.

필수

라이브 기본 흐름

방송 입장, 방송자 화면, 시청자 화면, 채팅 통합, 시청자 수 표시를 우선합니다.

필수

투표와 권한

투표 생성과 참여, 결과 반영, 방송자와 시청자 권한 분리를 구현합니다.

선택

후원과 채널 확장

Toss Payments, 공개 채널, VOD, 매니저 기능은 발표 범위에 맞춰 후속으로 둡니다.

MVP 필수 기능

라이브 기능이라고 말하기 위해 반드시 보여야 하는 범위입니다.

기능연결 라우터접근비고
라이브 목록/live공개Anon User도 현재 방송 중인 목록을 볼 수 있습니다.
라이브 시청/live/[creatorId]공개채팅 읽기와 방송 시청은 공개, 입력과 참여는 로그인 필요로 둡니다.
방송 운영/channel/live본인만방송 준비, 시작, 종료, 현재 상태 표시를 처리합니다.
방송 설정/channel/broadcast본인만방송 제목, 공개 상태, 다시보기 저장 여부, 스트림 키 표시를 관리합니다.
실시간 채팅 통합/live/[creatorId]입력은 로그인 필요기존 채팅 기능을 라이브 화면에 맞게 재사용합니다.
투표/live/[creatorId], /channel/live참여는 로그인 필요방송자는 생성, 시청자는 참여, 결과는 실시간 반영합니다.
시청자 수/live/[creatorId], /channel/analytics시청 수 공개라이브 화면과 통계 화면에서 같은 기준으로 보여줍니다.

선택 기능

MVP 이후 기능은 라우터 예약과 데이터 구조만 고려합니다.

기능라우터 후보접근비고
후원금 충전/user/donations/charge로그인 필요Toss Payments를 붙일 때 활성화합니다.
후원 내역/user/donations/history로그인 필요충전, 사용, 보낸 후원을 type query로 필터링합니다.
공개 채널 홈/channel/[creatorId]선택 기능SOOP 방송국 같은 팬카페형 페이지로 확장할 때 사용합니다.
공개 게시판/channel/[creatorId]/board선택 기능채널 기능 자체가 확정된 뒤 추가합니다.
다시보기/channel/[creatorId]/vod선택 기능방송 종료 후 자동 저장 옵션과 연결합니다.
매니저/channel/managers 후보선택 기능MVP에서는 크리에이터 본인만 방송 운영합니다.
블랙리스트/channel/chat 내부 탭 후보선택 기능강퇴와 영구 차단은 분리해서 나중에 설계합니다.
방송 분석 리포트/channel/analytics선택 기능실시간 통계 이후 종료 리포트로 확장합니다.
