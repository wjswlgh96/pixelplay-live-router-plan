Design Preview · MVP

디자인 프리뷰는 MVP 필수 라우터만 남기고, 라이브 중심 흐름을 먼저 확인할 수 있게 정리했습니다.

본문 순서는 공개 랜딩, 라이브 탐색, 라이브 시청, 방송 운영, 채팅 설정, 보안 설정, 실시간 통계, 채팅, 프로필, 후원 순서입니다. 탐색은 팔로우·인기·방금 시작·채팅 활발 같은 필터로 정리하고, 후원과 실시간 통계는 MVP에 포함합니다. 채널 홈 같은 후속 후보는 라우터 문서와 로드맵에서만 확인합니다.

1공개 랜딩은 별도 문서에서 확인

인덱스 페이지의 디자인 프리뷰는 현재 구현된 랜딩과 혼동되지 않도록 이 문서에서 제거했습니다. 공개 랜딩의 최신 구조, CTA, 애니메이션 방향은 별도 랜딩 페이지 문서에서 확인합니다.

참고치지직 기준 채널 사이드바 플로우

MVP 기준 채널 관리는 치지직 스튜디오의 방송 관리 흐름만 가져오되, 방송 운영은 /channel/live, 채팅 설정은 /channel/chat, 보안 설정은 /channel/security, 통계는 /channel/analytics로 분리합니다. 방송 채팅 분리는 /live/[creatorId]/chat 팝아웃으로 처리하고, 후원은 /user/donations에서 관리합니다.

방송 운영/channel/live

방송 준비, 시작, 종료, 공지, 투표, 채팅 일시정지를 처리합니다.

채팅 설정/channel/chat

채팅 속도, 링크 차단, 금칙어를 관리합니다.

보안 설정/channel/security

스트림키와 OBS 채팅창 URL을 관리합니다.

실시간 통계/channel/analytics

방송 중 시청자, 채팅, 후원, 투표 지표를 확인합니다.

채팅 팝아웃/live/[creatorId]/chat

방송 채팅만 별도 창으로 띄우는 공개 보조 라우터입니다.

후원/user/donations

잔액, 충전 Dialog, 후원 내역 탭을 한 화면에서 관리합니다.

치지직 확인 결과채팅 팝아웃 URL

치지직 방송 운영 화면의 iframe은 /iframe/live/{creatorId}/chat이고, 팝업으로 열면 /live/{creatorId}/chat으로 이동합니다. PixelPlay는 내부 iframe route를 따로 노출하지 않고 공개 보조창만 /live/[creatorId]/chat으로 둡니다.

/channel/live 채팅 패널→/live/[creatorId]/chat팝아웃

참고탭 전환·다이얼로그 상태

화면별 기본 프리뷰에서 빠지기 쉬운 active, hover, open, confirm 상태를 한곳에 모았습니다. 실제 구현에서는 Shadcn/Base UI 컴포넌트를 쓰되, 시각 톤은 아래처럼 PixelPlay 민트와 라이브 코랄을 기준으로 통일합니다.

scopeCommon UI states
targets

Header, Select Tab, Filter Chip, Channel Sidebar, Donation Tab, Dialog, Dropdown, Popover.

rule

탭은 현재 화면의 맥락을 먼저 보여주고, 위험 액션은 반드시 확인 Dialog를 거칩니다.

tone

라이트는 선명한 SaaS 톤, 다크는 방송 운영 콘솔 톤으로 맞춥니다.

ACTIVE 선택된 탭은 한 번에 보이게

민트 배경, 굵은 텍스트, 낮은 그림자로 현재 위치를 확실히 구분합니다.

LIVE ACTION 라이브 액션은 코랄로 제한

방송 시작, 라이브 입장, 후원처럼 행동성이 강한 버튼에만 사용합니다.

DIALOG 중요 액션은 짧고 명확하게

제목, 한 줄 설명, 핵심 요약, 취소·확인 버튼만 남겨 판단을 빠르게 만듭니다.
