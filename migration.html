Migration Notes

현재 라우터는 redirect 없이 삭제 기준으로 정리합니다.

아직 외부 사용자가 없고 발표 링크도 새로 정리할 수 있다는 전제에서, 과거 라우터 호환보다 개발 단순성을 우선합니다.

삭제와 대체 라우터

배포 전 빠르게 정리할 수 있는 항목입니다.

기존 라우터처리대체 라우터접근비고
/화면 역할 변경/공개기존 채팅과 라이브 탭 전환 홈을 공개 랜딩으로 전환합니다.
/chat-room/[roomId]삭제/chat/room/[roomId]로그인 필요room 세그먼트는 단수로 고정합니다.
/search/chat삭제/chat/search로그인 필요채팅 도메인 내부 검색으로 이동합니다.
/profile삭제/user/profile로그인 필요유저 사이드바 확장을 고려해 root를 user로 변경합니다.
/user신규 기본 이동/user/profile로그인 필요빈 root 페이지를 만들지 않습니다.

구현 순서

라우터 정리 후 라이브 기능을 붙이는 순서입니다.

1단계

인덱스 랜딩 전환과 Header CTA를 먼저 반영합니다. 방송하기는 로그인 시 /channel/live, 비로그인 시 /auth/login?next=/channel/live로 보냅니다.

2단계

채팅 라우터를 /chat, /chat/room/[roomId], /chat/search로 정리합니다. 기존 라우터는 redirect 없이 삭제합니다.

3단계

프로필 라우터를 /user/profile로 이동하고, 후원 확장을 위해 /user/donations 구조를 예약합니다.

4단계

라이브 공개 라우터 /live, /live/[creatorId], /live/search를 만들고 시청자 화면을 먼저 완성합니다.

5단계

크리에이터 방송 운영 라우터 /channel/live, /channel/broadcast, /channel/chat을 붙입니다.

권한 원칙

프론트 라우터 이름보다 서버 권한과 RLS가 실제 보안 경계입니다.

  • 공개 시청은 허용합니다. 비로그인 사용자는 라이브 목록과 라이브 시청 페이지에 들어올 수 있습니다.
  • 상호작용은 로그인 이후에 허용합니다. 채팅 입력, 투표 참여, 팔로우, 후원, 고화질 기능은 로그인 필요 상태로 처리합니다.
  • 채널 관리 라우터는 userId를 받지 않습니다. /channel/* 서버 로직은 현재 로그인한 유저 기준으로만 데이터를 조회하고 수정합니다.
  • 공개 식별자는 creatorId만 노출합니다. /live/[creatorId]와 후속 공개 채널 기능에서만 URL 식별자를 사용합니다.
