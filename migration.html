Migration Notes

현재 라우터는 치지직형 웹 스튜디오 흐름으로 정리합니다.

과거 초안의 임의 통합 설정 라우터는 제거하고, 정보량이 많은 채널 관리는 /channel/live, /channel/chat, /channel/security, /channel/analytics로 나눕니다. 채팅 팝아웃과 후원 Dialog는 별도 흐름으로 둡니다.

삭제와 대체 라우터

배포 전 빠르게 정리할 수 있는 항목입니다.

/화면 역할 변경/공개기존 채팅과 라이브 탭 전환 홈을 공개 랜딩으로 전환합니다.

구현 순서

라우터 정리 후 라이브 기능을 붙이는 순서입니다.

1단계

인덱스 랜딩 전환과 Header CTA를 먼저 반영합니다. 방송하기는 로그인 시 /channel/live, 비로그인 시 /auth/login?next=/channel/live로 보냅니다.

2단계

채팅 라우터를 /chat, /chat/room/[roomId], /chat/search로 정리합니다. 기존 라우터는 redirect 없이 삭제합니다.

3단계

프로필 라우터를 /user/profile로 이동하고, 후원은 /user/donations 한 화면 안의 Dialog와 탭으로 정리합니다.

4단계

라이브 공개 라우터 /live, /live/[creatorId], /live/[creatorId]/chat를 만들고 시청자 화면과 채팅 팝아웃을 먼저 완성합니다.

5단계

크리에이터 관리는 치지직 스튜디오처럼 /channel/live, /channel/chat, /channel/security, /channel/analytics를 사이드바로 이동하게 구성합니다.

라우터 작업 준비 체크리스트

dev에 반영하기 전에 삭제, 생성, 연결 순서를 한 번에 확인합니다.

삭제 대상

/chat-room/[roomId], /search/chat, /profile, /user/donations/charge, /user/donations/history, /channel/broadcast, /channel/prohibit-words는 라우터로 남기지 않습니다.

생성 대상

/live/[creatorId], /live/[creatorId]/chat, /channel/live, /channel/chat, /channel/security, /channel/analytics, /user/donations를 최종 라우터 기준으로 맞춥니다.

연결 대상

Header 방송하기 버튼은 /channel/live, UserAccountMenu 후원은 /user/donations, 채널 사이드바는 방송 운영, 채팅 설정, 보안 설정, 실시간 통계 순서로 연결합니다.

권한 대상

/live 하위 공개 시청 라우터는 비로그인 접근을 허용하고, /channel/*와 /user/*는 현재 로그인한 사용자 기준으로만 접근시킵니다.

권한 원칙

프론트 라우터 이름보다 서버 권한과 RLS가 실제 보안 경계입니다.

  • 공개 시청은 허용합니다. 비로그인 사용자는 라이브 목록, 라이브 시청 페이지, 채팅 팝아웃 읽기 화면에 들어올 수 있습니다.
  • 상호작용은 로그인 이후에 허용합니다. 채팅 입력, 투표 참여, 팔로우, 후원, 고화질 기능은 로그인 필요 상태로 처리합니다.
  • 채널 관리 라우터는 userId를 받지 않습니다. /channel/* 서버 로직은 현재 로그인한 유저 기준으로만 데이터를 조회하고 수정합니다.
  • 공개 식별자는 creatorId만 노출합니다. /live/[creatorId]와 /live/[creatorId]/chat에서만 URL 식별자를 사용합니다.
