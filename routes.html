Route Map

라우터는 역할, 접근, query, 비고 기준으로 고정합니다.

Anon User에게 공개되는 라우터와 로그인 이후 가능한 라우터를 명확히 나눕니다. 공개 시청과 인증 기반 상호작용을 분리하는 것이 핵심입니다.

공개 라우터

비로그인 사용자가 접근 가능한 화면입니다.

라우터역할접근query비고
/공개 랜딩. 채팅 시작하기, 라이브 둘러보기, 방송하기 CTA 제공.공개없음기존 탭형 인덱스를 대체합니다.
/live현재 송출 중인 라이브 목록.공개?sort=popular 후보src/components/live/live-list.tsx는 이 화면의 시청자 UI로 사용합니다.
/live/[creatorId]특정 크리에이터의 현재 라이브 시청.공개없음채팅 입력, 투표, 후원, 고화질 선택은 로그인 필요 상태로 제한합니다. 내부 식별자는 현재 public.user.id를 재사용할 수 있지만 라우터 param 이름은 creatorId로 표현합니다.
/live/search라이브 방송 검색.공개?query=검색어라이브 목록과 검색 결과를 분리해 공유 링크를 명확히 합니다.

채팅 라우터

채팅은 라이브와 별도 도메인으로 유지합니다.

라우터역할접근query비고
/chat채팅방 목록. 탭별 query result 기준으로 목록 표시.로그인 필요?tab=recent 후보기존 인덱스 내부 탭에서 독립 라우터로 분리합니다.
/chat/room/[roomId]채팅방 상세.로그인 필요없음기존 /chat-room/[roomId] 삭제 후 대체합니다. 라우터 세그먼트는 단수 room을 사용합니다.
/chat/search채팅방 검색.로그인 필요?query=검색어기존 /search/chat 삭제 후 대체합니다.

유저 라우터

프로필, 팔로우, 후원금 충전과 사용 내역은 유저 영역에서 관리합니다.

라우터역할접근query비고
/user유저 영역 root.로그인 필요없음빈 화면으로 두지 않고 /user/profile로 기본 이동합니다.
/user/profile내 프로필 설정.로그인 필요없음기존 /profile 삭제 후 대체합니다.
/user/follows팔로우 중인 크리에이터와 채널 목록.로그인 필요?tab=channels 후보선택 기능이지만 유저 사이드바 확장을 위해 라우터를 예약합니다.
/user/donations후원 홈. 후원금 잔액, 충전 CTA, 최근 내역 요약.로그인 필요없음/user/wallet은 사용하지 않고 donations로 통일합니다.
/user/donations/charge후원금 충전.로그인 필요?amount=금액 후보Toss Payments 연동 시 결제 시작 화면이 됩니다.
/user/donations/history충전, 사용, 보낸 후원 내역.로그인 필요?type=charge|use|sent별도 /user/donations/sent를 만들지 않고 history 필터로 처리합니다.

채널 관리 라우터

크리에이터 본인의 방송 운영과 채널 설정 영역입니다.

라우터역할접근query비고
/channel내 채널 관리 홈.본인만없음현재 로그인한 유저 기준으로 조회합니다. URL에 userId를 받지 않습니다.
/channel/live방송 준비, 진행, 종료 관리.본인만없음방송 운영 페이지입니다. 시청자 페이지의 채팅 관리와 분리합니다.
/channel/analytics실시간 통계 대시보드.본인만?range=today 후보시청자 수, 채팅량, 투표 참여, 후원 요약을 표시합니다.
/channel/broadcast방송 기본 설정.본인만없음방송 제목 기본값, 다시보기 저장 여부, 지연 시간, 스트림 키를 관리합니다.
/channel/chat채팅 설정과 금칙어.본인만없음슬로우 모드, 금칙어, 메시지 강조 같은 운영 정책을 관리합니다.
/channel/profile채널 프로필 설정.본인만없음공개 채널 기능을 나중에 만들 때 소개, 배너, 링크의 기준 데이터가 됩니다.

인증과 API 후보

기존 인증 라우터는 유지하고 결제 API는 선택 기능 범위에 둡니다.

라우터역할접근query비고
/auth/login로그인.공개?next=이동경로채팅 시작하기와 방송하기 CTA에서 next를 전달합니다.
/auth/signup회원가입.공개없음기존 인증 흐름을 유지합니다.
/auth/complete-profile프로필 완성.로그인 필요없음신규 유저 온보딩 흐름입니다.
/auth/callbackOAuth 콜백.공개Provider 전달값기존 인증 콜백을 유지합니다.
/api/payments/toss/confirmToss Payments 결제 승인.서버 API없음후원금 충전 선택 기능에서 사용합니다.
/api/payments/toss/webhookToss Payments 이벤트 수신.서버 API없음결제 상태 동기화와 실패 복구에 사용합니다.
