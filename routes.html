MVP Router Scope

지금 개발할 라우터는 라이브 MVP에 필요한 것만 남깁니다.

최종 후보를 전부 만들지 않고, 발표와 실제 개발 흐름에 필요한 필수 라우터만 우선 생성합니다. 후원과 실시간 통계는 MVP에 포함하고, 알림, 리허설, 공개 채널, 매니저 기능은 후속 후보로 분리합니다.

Public

시청자 진입

랜딩, 라이브 목록, 라이브 시청, 채팅 팝아웃을 공개 라우터로 둡니다.

Creator

채널 관리 분리

방송 운영, 채팅 설정, 보안 설정, 실시간 통계를 각각 분리해 정보량을 낮춥니다.

Donation

후원 MVP

후원 홈, 충전 Dialog, Toss 결제 승인·웹훅까지 MVP에 포함합니다.

Existing

채팅 유지

이미 구현된 채팅 목록, 상세, 검색 라우터는 깨지지 않게 유지합니다.

MVP 필수 라우터

이번 단계에서 실제 라우터 파일과 네비게이션 연결 대상으로 보는 범위입니다.

라우터역할접근query비고
/공개 랜딩.공개없음로그인 CTA 기본 next는 /live입니다.
/live현재 송출 중인 라이브 목록.공개없음src/components/live/live-list.tsx는 시청자 UI로 유지합니다.
/live/[creatorId]특정 크리에이터의 현재 라이브 시청.공개없음시청과 채팅 읽기는 공개, 채팅 입력과 투표 참여는 로그인 이후 허용합니다.
/live/[creatorId]/chat라이브 채팅 팝아웃 전용 보조창.공개 읽기없음치지직의 /live/{creatorId}/chat 팝업 흐름을 따릅니다.
/channel/live방송 운영.본인만없음방송 준비, 시작, 종료, 송출 미리보기, 공지, 투표, 채팅 일시정지처럼 방송 중 바로 쓰는 기능만 둡니다.
/channel/chat채팅 설정과 금칙어.본인만없음채팅 속도, 링크 차단, 이모티콘 모드, 금칙어를 관리합니다.
/channel/security방송 보안 설정.본인만없음스트림키와 OBS Studio 채팅창 URL을 관리합니다. 민감 정보 표시는 재확인 Dialog를 거칩니다.
/channel/analytics실시간 통계.본인만없음방송 중 시청자 수, 채팅량, 후원, 투표 참여율을 확인합니다.
/chat채팅방 목록.로그인 필요?tab=... 후보이미 구현된 채팅 목록을 독립 라우터로 유지합니다.
/chat/room/[roomId]채팅방 상세.로그인 필요없음기존 /chat-room/[roomId]를 대체합니다.
/chat/search채팅방 검색.로그인 필요?query=검색어기존 /search/chat를 대체합니다.
/user/profile내 프로필 설정.로그인 필요없음아바타 메뉴와 /user 기본 이동 대상입니다.
/user/donations후원 홈.로그인 필요?tab=... 후보잔액, 충전 Dialog, 충전·사용·보낸 후원 내역을 한 화면에서 처리합니다.
/api/payments/toss/confirmToss 결제 승인 API.서버 처리없음후원금 충전 Dialog에서 결제 성공 후 호출합니다.
/api/payments/toss/webhookToss 결제 웹훅 API.서버 처리없음결제 상태 보정과 실패·취소 이벤트 처리를 맡깁니다.
/auth/login로그인.공개?next=이동경로next가 없거나 잘못되면 /live로 보냅니다.
/auth/signup회원가입.공개없음기존 인증 흐름을 유지합니다.
/auth/callbackOAuth 콜백.공개Provider 전달값기존 인증 콜백을 유지합니다.

MVP에서 제외한 후속 후보

라우터 파일을 먼저 만들지 않고, 필요해지는 시점에 추가합니다.

후보라우터 후보판단이유
라이브 검색 전용 페이지/live/search후속MVP에서는 /live 안의 검색 UI로 충분합니다.
채널 관리 홈/channel후속방송 운영과 설정이 먼저입니다. 홈 대시보드는 여유가 있을 때 붙입니다.
알림/channel/notifications후속치지직 플로우 참고 대상이지만 MVP 기능 증명에는 필수는 아닙니다.
리허설 방송/channel/rehearsal후속정식 송출 전 테스트 방송은 선택 기능입니다.
시청자 관리/channel/viewers후속강퇴와 제한 관리는 라이브 채팅 UI에서 최소 처리 후 확장합니다.
콘텐츠 관리/channel/content후속VOD, 클립, 다시보기 관리는 방송 종료 후 기능으로 미룹니다.
수익 관리/channel/revenue후속후원은 MVP에 포함하고, 정산과 크리에이터 수익 리포트는 MVP 이후 결제 안정화 단계에서 진행합니다.
채널 프로필/channel/profile후속공개 채널 기능이 확정될 때 추가합니다.
권한 관리/channel/permissions후속MVP 방송 운영은 크리에이터 본인만 수행합니다.
팔로우/user/follows후속라이브 목록과 시청이 먼저입니다.

이 문서는 MVP 착수용 라우터 기준입니다. 후속 후보는 삭제가 아니라 개발 순서에서 뒤로 미룬 항목입니다.
