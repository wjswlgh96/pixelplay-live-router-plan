# 다음 프로젝트 라이브 구현 작업

## 먼저 연결할 라우터와 메뉴

- `/live/search`를 공개 검색 페이지로 구현합니다.
- `/user/follows`를 사용자 사이드바에 연결합니다.
- `/channel/live`, `/channel/chat`, `/channel/security`, `/channel/analytics`를 채널 사이드바에 연결합니다.
- UserAccountMenu에는 프로필 설정, 채널 관리, 팔로우, 후원, 비밀번호 변경, 로그아웃 흐름을 정리합니다.
- 채널 사이드바 하단에는 프로필 설정과 로그아웃을 별도로 둡니다.

## DB 확정 전에도 가능한 UI 작업

- `/live/search` 데모 UI를 치지직 검색 결과 흐름처럼 구성합니다.
- `/user/follows` 데모 UI를 팔로우한 크리에이터 목록과 LIVE 상태 표시 중심으로 구성합니다.
- `/channel/security`에는 스트림키 표시, 재발급, OBS 채팅창 URL 복사 Dialog를 먼저 UI로 만듭니다.
- `/channel/chat`에는 채팅 참여 범위, 본인인증, 이모티콘 모드, 저속 모드, 금칙어 관리 UI를 만듭니다.
- `/user/donations`에는 후원금 충전 Dialog를 둡니다.

## DB 확정 이후 작업

- `creator_follow` 테이블과 RLS를 추가합니다.
- 라이브 세션, 라이브 채팅 메시지, 라이브 시청자 presence, 투표, 후원 ledger를 확정합니다.
- Toss Payments 승인 API와 웹훅 API를 결제 상태 보정까지 포함해 구현합니다.
- 스트림키는 DB에 평문 저장하지 않고 별도 암호화 또는 서버 전용 처리 정책을 확정합니다.

## 주의할 점

- 일반 채팅방 기능을 라이브 채팅에 억지로 재사용하지 않습니다.
- 라이브 전용 채팅은 방송 세션, 크리에이터, 시청자 presence에 맞춰 새로 설계합니다.
- 방송 운영 페이지는 크리에이터 본인만 접근합니다.
- 채팅 관리자는 일반 시청자용 라이브 페이지에서 유저 클릭 액션으로 관리하는 방향을 우선합니다.
- 실제 스트림키 표시와 재발급은 구현 전 재확인 Dialog와 RLS를 먼저 확정합니다.
