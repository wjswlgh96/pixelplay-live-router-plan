The Playground for Live Interaction

채팅과 라이브를 분리하고, 크리에이터 운영 흐름은 채널 관리로 모읍니다.

이 문서는 PixelPlay의 라이브 기능 개발을 시작하기 전 라우터, 접근 정책, 현재 프로젝트 변경 범위, 기본 화면 디자인 방향을 고정하기 위한 HTML 기획서입니다.

공개 진입비로그인 시청 허용

랜딩, 라이브 둘러보기, 라이브 시청은 공개 흐름으로 두고 채팅 입력, 후원, 고화질 같은 상호작용만 로그인 이후에 허용합니다.

//live/live/[creatorId]
라우터 분리채팅과 라이브 독립 진입

채팅 목록과 라이브 목록은 인덱스 탭 전환이 아니라 각각의 URL을 가진 독립 라우터로 나눕니다.

/chat/live
크리에이터 운영내 채널 관리로 통합

방송 준비, 진행, 종료, 설정은 현재 로그인한 유저 기준의 채널 관리 영역에서 처리합니다.

/channel/*

핵심 결정

라우터 이름과 접근 정책에서 흔들리면 구현 중 구조가 다시 갈라지므로 먼저 고정합니다.

  • 인덱스는 랜딩으로 전환합니다. 기존 탭형 홈은 제거하고, 채팅 시작하기와 라이브 둘러보기, 방송하기 CTA를 명확히 분리합니다.
  • 후원 도메인은 donations로 통일합니다. /user/wallet은 사용하지 않고 /user/donations 한 화면에서 잔액, 충전 Dialog, 후원 내역을 처리합니다.
  • 채널 관리는 치지직형 웹 스튜디오 흐름을 따릅니다. MVP 사이드바에는 실제 구현할 방송 운영, 채팅 설정, 보안 설정, 실시간 통계만 두고 후속 후보는 라우터 문서로 분리합니다.
  • 크리에이터 운영 화면은 /channel/live입니다. 시청자용 라이브 목록 컴포넌트와 방송 운영 페이지를 혼동하지 않도록 라우터를 분리하고, 채팅창 분리는 /live/[creatorId]/chat 팝아웃 URL로 처리합니다.
  • 비로그인 시청은 허용하고 상호작용은 제한합니다. 라이브 보기 자체는 공개하되 채팅 입력, 투표, 팔로우, 후원, 고화질 기능은 로그인 이후에 허용합니다.
  • 라이브 공개 URL은 creatorId로 둡니다. 내부 값은 현재 public.user.id를 재사용할 수 있지만 URL 의미는 유저 계정이 아니라 크리에이터 방송 진입점으로 표현합니다.
  • 채널 관리는 MVP 라우터를 분리합니다. 방송 운영은 /channel/live, 채팅 설정은 /channel/chat, 스트림키와 OBS 채팅창 URL은 /channel/security, 실시간 통계는 /channel/analytics로 나눕니다.
  • 팔로우는 MVP에 포함합니다. 팔로우 대상은 공개 채널 게시판이 아니라 크리에이터이며, /user/follows에서 다시 볼 수 있게 합니다.
